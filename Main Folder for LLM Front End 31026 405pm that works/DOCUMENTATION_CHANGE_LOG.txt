Fix: post-tool grounding message inserted before tool results, breaking citations:
- Root cause: in _chat_with_optional_tools (services/ollama_client.py), the
  post-tool grounding system message was appended to augmented_messages
  immediately after the assistant tool-call message and BEFORE the tool result
  messages. The Ollama tool-use protocol requires the sequence:
    [assistant]  tool_calls=[search_web]
    [tool]       result JSON
    [system]     post-tool grounding

  models broke the assistant→tool pairing and ignored the tool result. The model
  never saw the source_id values in the results, so it produced no [S1]-style
  citations. _build_validated_sources_from_answer found nothing in the answer

  blank.

  after the for-loop that appends all tool result messages, so the final message
