FILE INTERACTION LOG
====================

Purpose
-------
This document maps each repository file to its confirmed responsibility after a
file-by-file code-to-documentation conformance audit. The goal is traceability:
someone reviewing the project should be able to see where behavior lives before
reading every line of code.

Repository File Map
-------------------

Top-Level Runtime Files
~~~~~~~~~~~~~~~~~~~~~~~
`main.py`
    Entry point. Creates `LocalLLMFrontendApp` and runs the UI loop.

`app.py`
    Composition root. Instantiates ConfigService, SessionService, OllamaClient,
    ChatController, and MainWindow. Attaches the controller to the view and
    bootstraps initial UI state.

`run_local_llm_frontend.sh`
    Shell launcher. Changes into the project directory, checks for `python3`,
    and starts `main.py`.

`requirements.txt`
    Third-party dependency list. Current package requirements: `beautifulsoup4` and optional `sentence-transformers` support for the local cross-encoder reranker backend.

Documentation Files
~~~~~~~~~~~~~~~~~~~
`README.txt`
    High-level architecture, runtime behavior, dependencies, launch guide, and
    current description of the two-stage search reranking pipeline.

`FILE_LOG.txt`
    File-role map for the repository.

`DOCUMENTATION_CHANGE_LOG.txt`
    Log of what documentation was rewritten after this audit pass.

`CODE_DOC_CONFORMANCE_AUDIT.txt`
    File-by-file audit record describing code/doc agreement, mismatches found,
    validation actions taken, and resulting documentation updates.

`TEST_REPORT.txt`
    Summary of the current automated validation result bundled with this package.

Controller Package
~~~~~~~~~~~~~~~~~~
`controllers/__init__.py`
    Package marker and package-role explanation for the controller layer.

`controllers/chat_controller.py`
    Main orchestration file. Owns active runtime session state, mediates between
    view and services, creates frozen GenerationJob snapshots, starts background
    reply generation, streams chunks to the UI, persists sessions, blocks
    conflicting actions during generation, handles model refresh, and exposes
    session-management operations.

Model Package
~~~~~~~~~~~~~
`models/__init__.py`
    Package marker and package-role explanation for structured data models.

`models/chat_message.py`
    Dataclass for one message. Stores role, content, timestamp, and optional
    thinking text. Includes serialization helpers.

`models/conversation_session.py`
    Dataclass for one saved conversation session. Stores session metadata and
    a list of ChatMessage objects. Includes serialization helpers.

`models/generation_job.py`
    Dataclass for one in-flight request snapshot. Stores the session ID, model,
    runtime config snapshot, message history snapshot, web-search settings
    including reranker options and `web_max_pages`, and request lifecycle
    fields such as accumulated output and error text.

Service Package
~~~~~~~~~~~~~~~
`services/__init__.py`
    Package marker and package-role explanation for service-layer code.

`services/config_service.py`
    Configuration persistence and normalization. Creates `config.json` when
    missing, repairs invalid config values by falling back to safe defaults,
    writes updates atomically, and stores both the dedicated search-reranker
    settings and the web-fetch safety policy settings used by the retrieval
    layers.

`services/session_service.py`
    Session persistence layer. Creates new session objects, saves JSON files,
    loads sessions, lists sessions sorted by recency, deletes sessions, builds
    human-readable filenames, and removes duplicate old files for renamed
    sessions.

`services/search_service.py`
    SearXNG client. Normalizes instance base URLs, performs searches, maps raw
    result records into a stable shape, and offers a connection test with
    actionable error messages.

`services/reranker_service.py`
    Dedicated search-result reranker layer. Provides the deterministic stage-1
    lexical reranker, the optional Ollama stage-2 reranker, and the optional
    sentence-transformers cross-encoder stage-2 reranker. Rebuilds ranked
    payloads while preserving candidates omitted by backend output.

`services/web_fetch_service.py`
    URL fetch helper. Downloads page HTML, blocks localhost/private/internal
    targets, resolves hostnames before fetch, re-validates redirects, enforces
    content-type and size limits, applies optional domain allow/block rules,
    and extracts a readable title/body text payload for the model.

`services/ollama_client.py`
    Ollama HTTP client. Sends streamed chat requests, lists models, tests
    connectivity, requests model unload, builds tool schemas, executes
    `search_web` and `fetch_url_content`, optionally routes search results
    through the configured stage-2 reranker backend before exposing them to the
    main model,
    passes the saved fetch-safety policy into WebFetchService, enforces the
    per-request `web_max_pages` fetch cap, injects post-tool grounding, retries
    with forced search in some current-information scenarios, and computes final
    throughput metrics from stream metadata.

`services/gpu_monitor_service.py`
    AMD GPU metric reader. Prefers `amd-smi`, falls back to `rocm-smi`, parses
    multiple output layouts, computes VRAM percentage when only used/total are
    available, and returns normalized text values for the UI.

UI Package
~~~~~~~~~~
`ui/__init__.py`
    Package marker and package-role explanation for Tkinter UI files.

`ui/main_window.py`
    Main application window. Builds the sidebar, session manager, transcript,
    thinking panel, input controls, status bar, model dropdown, GPU live
    readout, token meter, main-window internet-search toggle, Ctrl+wheel text
    zoom bindings, and the vertical pane layout that lets the user resize the
    transcript, thinking, and input regions. Handles UI-thread-safe updates and
    background GPU polling integration.

`ui/settings_window.py`
    Settings popup. Edits Ollama and SearXNG-related settings, exposes the
    dedicated search-reranker controls, shows the user-visible fetch policy
    controls, tests connectivity, refreshes installed model choices, handles
    model selection, and routes the VRAM flush action to the controller.

Utility Package
~~~~~~~~~~~~~~~
`utils/__init__.py`
    Package marker and package-role explanation for utility helpers.

`utils/threading_helpers.py`
    Small helper that starts daemon background threads for non-UI work.

Test Files
~~~~~~~~~~
`tests_web_search_integration.py`
    Automated tests for search normalization, dedicated reranking behavior, URL
    repair, fetch extraction, fetch safety policy enforcement, GenerationJob web
    config propagation, runtime `web_max_pages` enforcement, tool-guidance
    injection, post-cutoff grounding injection, forced-search behavior, and one
    opt-in live SearXNG -> rerank -> fetch smoke test that emits step-by-step
    trace logs.

`tests_test_generation_job_flow.py`
    Automated tests for session-binding correctness during in-flight
    generation, model snapshot correctness, unload routing, config stability,
    UI shutdown safety, GPU metric parsing, and throughput metric handling.

Generated / Runtime Files and Directories
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
`config.json`
    Created at runtime. Stores normalized app settings.

`sessions/`
    Created at runtime. Stores per-session conversation JSON files.

`__pycache__/`
    Python bytecode cache directory. Not source of truth for behavior or docs.

How the Files Interact
----------------------
1. `main.py` starts `LocalLLMFrontendApp` from `app.py`.
2. `app.py` constructs services and the controller, then creates the main UI.
3. `ui/main_window.py` captures user actions and forwards them to
   `controllers/chat_controller.py`.
4. `controllers/chat_controller.py` reads/writes settings through
   `services/config_service.py`, including direct persistence of the main-window
   internet-search toggle, session state through `services/session_service.py`,
   and model responses through `services/ollama_client.py`.
5. `services/ollama_client.py` optionally calls `services/search_service.py`,
   `services/reranker_service.py`, and `services/web_fetch_service.py` during
   the tool-calling flow.
6. `ui/main_window.py` polls GPU metrics through the controller, which reads
   them from `services/gpu_monitor_service.py`.
7. `models/*.py` define the structured data objects shared across the layers.


Adversarial Review Follow-Up Notes
----------------------------------
- `services/ollama_client.py` now preserves Ollama-compatible tool message
  ordering, propagates `tool_call_id`, narrows forced-search fallback
  heuristics, and shares one request runtime state across both normal and
  fallback search execution.
- `services/web_fetch_service.py` now enforces the incremental response byte
  cap exactly at the configured limit.
- `services/session_service.py` now loads one requested session directly rather
  than scanning every session file on disk.
- `ui/main_window.py` now resets thinking-panel visibility during session
  switches, avoids redundant show toggles while loading history, and clears the
  GPU inflight flag when UI shutdown prevents callback scheduling.
- `controllers/chat_controller.py` now auto-titles only the exact generated
  default session-title pattern, which prevents accidental overwrite of manual
  titles that happen to begin with `Session `.
